# Security Policy

## Supported Versions

Security fixes are prioritized for the latest released version on the default branch.

## Reporting a Vulnerability

Please do **not** open public issues for security-sensitive reports.

Use one of the following:

- GitHub Security Advisory ("Report a vulnerability")
- Direct maintainer contact (if previously established)

Include:

- A clear description of the issue
- Reproduction steps
- Impact assessment
- Any proof-of-concept details

## Response Expectations

- Initial triage target: within 7 days
- Status updates: provided as fixes are validated
- Public disclosure: coordinated after a patch or mitigation is available
